## 2. 개요

### 2.1 배경 및 목적
- IT 회사 내 프로젝트 관리의 효율성 향상
- 프로젝트 진행 상황의 가시성 확보
- 간트 차트를 통한 일정 관리 지원
- 웹 및 모바일 환경에서의 접근성 제공

### 2.2 범위
- 프로젝트 정보 등록, 수정, 삭제, 조회 기능
- 사용자 (PM) 및 관리자 (임원/PMO) 역할 기반 접근 제어
- 웹 및 모바일 플랫폼 지원
- 간트 차트 기반 프로젝트 일정 시각화

### 2.3 대상 사용자
1. **사용자 (PM)**: 프로젝트 정보 입력 및 관리
2. **관리자 (임원/PMO)**: 등록 프로젝트 및 일정 관리 감시

## 3. 사용자 스토리

### 3.1 PM (프로젝트 관리자)
- US-001:作為 PM, 나는 프로젝트를 등록할 수 있어야 한다
- US-002:作为 PM,我是 등록한 프로젝트를 수정/삭제할 수 있어야 한다
- US-003:作为 PM,我是 등록된 프로젝트 목록을 조회할 수 있어야 한다
- US-004:作为 PM,我是 간트 차트를 통해 프로젝트 일정을 시각화할 수 있어야 한다
- US-005:作为 PM,我是 모바일 환경에서도 프로젝트 정보를 관리할 수 있어야 한다

### 3.2 관리자 (임원/PMO)
- US-010:作为 管理者,我是 전체 프로젝트 현황을 한눈에 조회할 수 있어야 한다
- US-011:作为 管理者,我是 프로젝트 일정을 모니터링하고 관리할 수 있어야 한다
- US-012:作为 管理者,我是 프로젝트 진행 상태를 확인하고 보고서 생성이 가능해야 한다

## 5. 비기능 요구사항

### 5.1 성능
- 페이지 로딩 시간: 3 초 이내
- 동시 사용자 지원: 최소 100 명
- API 응답 시간: 200ms 이내 (95 번째 백분위)

### 5.2 보안
- 비밀번호 암호화 (bcrypt)
- JWT 기반 인증
- SQL Injection 방지
- XSS 방어

### 5.3 호환성
- 브라우저: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- 모바일: iOS 14+, Android 10+

### 5.4 유지보수성
- 코드 주석 및 docstring 준수
- 테스트 커버리지 80% 이상
- 모듈화된 구조

## 6. 기술 스택

### 6.1 백엔드
- Python 3.11+
- FastAPI
- SQLAlchemy (ORM)
- SQLite (개발), PostgreSQL (운영)
- Pytest (테스트)

### 6.2 프론트엔드
- HTML5, CSS3, JavaScript (ES6+)
- React 18+
- React Gantt (간트 차트 라이브러리)
- Tailwind CSS

### 6.3 인프라
- Docker 컨테이너링
- Nginx 웹 서버

## 7. 데이터 모델

### 7.1 사용자 (User)
```
- id: UUID (PK)
- email: string (unique)
- password_hash: string
- name: string
- role: enum (USER, ADMIN)
- created_at: datetime
- updated_at: datetime
```

### 7.2 프로젝트 (Project)
```
- id: UUID (PK)
- project_code: string (unique)
- project_name: string
- client: string (고객사)
- designer: string (설계자)
- developers: text (개발자 목록, JSON)
- start_date: date
- end_date: date
- status: enum (PLANNING, IN_PROGRESS, COMPLETED, ON_HOLD)
- created_by: UUID (FK → User)
- created_at: datetime
- updated_at: datetime
- deleted_at: datetime (nullable, soft delete)
```

## 8. API 설계 (RESTful)

### 8.1 사용자 API
```
POST   /api/v1/auth/login
POST   /api/v1/auth/logout
GET    /api/v1/auth/me
```

### 8.2 프로젝트 API
```
POST   /api/v1/projects
GET    /api/v1/projects
GET    /api/v1/projects/{project_id}
PUT    /api/v1/projects/{project_id}
DELETE /api/v1/projects/{project_id}
```

### 8.3 간트 차트 API
```
GET    /api/v1/gantt/projects
POST   /api/v1/gantt/schedule/update
```
